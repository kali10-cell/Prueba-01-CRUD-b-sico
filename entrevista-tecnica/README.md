# Entrevista Técnica — Guía de Preparación

Material para practicar entrevistas técnicas de desarrollo web con el stack: **Next.js, Supabase, Vercel y PWA**.

---

## Cómo usar este material

### Para el alumno
Estudia los archivos por tema antes de la clase. No memorices las respuestas — entiéndelas. En la entrevista real te harán preguntas de seguimiento y la respuesta memorizada no sirve.

### Para la dinámica en clase (simulacro de entrevista)
1. Se forman parejas: uno hace de entrevistador, otro de candidato
2. El entrevistador elige un rol de la carpeta `roles-entrevistador/`
3. Se hacen 5-6 preguntas del tema asignado (~15 min)
4. Se rota: el candidato pasa a entrevistar
5. El profesor detiene y debate cuando escucha una respuesta floja

**El entrevistador evalúa 3 cosas** (ver `../pruebas-tecnicas/RUBRICA.md`):
- Claridad de explicación
- Cómo gestiona las preguntas que no sabe
- Profundidad técnica

## Formato de cada pregunta

Cada pregunta sigue este formato:

```

**Respuesta esperada:** ...
**Señal de alerta:** ...
**Pregunta de seguimiento típica:** ...
```

## Consejo final

En una entrevista real, **no saber algo no es un problema**. El problema es fingir que sabes. Lee el archivo `07-como-gestionar-preguntas-dificiles.md` antes de cualquier simulacro.
