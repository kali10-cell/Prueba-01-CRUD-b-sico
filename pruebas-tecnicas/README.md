## Instrucciones para el alumno

### Puedes usar IA
Herramientas como ChatGPT, GitHub Copilot o similares están permitidas. Así funcionan las empresas hoy.

**Lo que se evalúa NO es si usaste IA, sino:**
- Que el resultado funciona
- Que el código es legible y tiene sentido
- Que puedes explicar cada decisión técnica
- Que tu README muestra que entiendes lo que has construido

Después de entregar la prueba habrá una **defensa técnica** de 15-20 minutos. Si no puedes explicar el código que has entregado, quedará claro.

### Lo que debes entregar
1. Enlace al repositorio de GitHub (público o compartido)
2. Enlace al deploy en Vercel (funcionando)
3. El archivo `README.md` del proyecto rellenado con la plantilla ([README-ALUMNO-TEMPLATE.md](README-ALUMNO-TEMPLATE.md))

### Consejos generales
- Lee el enunciado completo antes de empezar. Los criterios de evaluación están al final.
- Entrega algo que funcione antes que algo que esté completo al 100% pero roto.
- Comenta las decisiones que tomaste que no son obvias. No el qué, sino el por qué.
- Si no llegas a implementar algo, escríbelo en el README: qué harías y cómo.

## Para el profesor — Guías de solución

Cada prueba tiene un archivo `SOLUCION-GUIA.md` con:
- Arquitectura recomendada
- Puntos clave que el alumno debería haber resuelto
- **Preguntas de defensa** para la revisión oral
- Criterios de puntuación detallados

Los alumnos no tienen acceso a estos archivos.
